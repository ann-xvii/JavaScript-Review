Object-oriented programming

Javascript objects are very simple collections of name value pairs.

There are two ways of creating a simple object in Javascript:

  1. var obj = new Object();
  2. var obj = {};

We can also create an object entirely as follows:

obj = { name: { first: 'Gandalf', last: 'the Grey' }, address: 'Middle Earth' };

In OOP, an object is an instance of a class. A class defines the characteristics of the object.

Example:

This is how we declare a class that represents a book:

function Book(title, pages, isbn){
		this.title = title;
		this.pages = pages;
		this.isbn = isbn;
	}

To instantiate this class, we can use the following code: 

		var book = new Book('title', 'pag', 'isbn');

Then we can access its attributes and update them as follows:

		console.log(book.title);
		book.title = 'Generation P';
		// updates the value of the book title
		console.log(book.title); // outputs the updated value

A class can also contain methods(functions):

		Book.prototype.printTitle = function(){
		   console.log(this.title);
		};
		book.printTitle();

We can also declare functions directly inside the class definition as well:

		function Book(title, pages, isbn) {
			this.title = title;
			this.pages = pages;
			this.isbn = isbn;
			this.printIsbn = function(){
				console.log(this.isbn);
			}
		}

In the prototype example, the printTitle function is going to be shared between all instances, and only one copy is going to be created.

When we use class-based definition, each instance of the class will have its own copy of the functions.

Using the protoype method saves memory and processing cost in regards to assigning the function to the instance.

however, you can only declare public functions and properties using the prototype method.
With a class-based definition, you can declare private functions and properties and the other methods inside the class can also access them.

Whenever possible, use the prototype method, unless, you need to keep some properties and methods private.
